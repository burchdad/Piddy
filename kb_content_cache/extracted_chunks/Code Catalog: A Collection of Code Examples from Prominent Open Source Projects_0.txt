viewBox="0 0 16 16" aria-hidden="true"><use xlink:href="#svg-link"></use></svg></a> FAQ </h1> <h3 id="why-do-we-need-this"> <a href="#why-do-we-need-this" class="anchor-heading" aria-labelledby="why-do-we-need-this"><svg viewBox="0 0 16 16" aria-hidden="true"><use xlink:href="#svg-link"></use></svg></a> Why do we need this? </h3> <p>When you ask someone how to get better at coding, the second most common response - right after writing more code - is to read other people’s code and learn from it.</p> <p>While it sounds very reasonable, it’s hard to implement it in practice. So you find the source for your favorite software on GitHub and start reading it. Firstly, for any established and mature product, the codebase is probably huge, complex and very hard to get started with. It’s unlikely that the design is documented anywhere. Even if the high-level architecture is documented, the structure of the codebase almost certainly isn’t. It probably depends on some libraries and frameworks you’ve never heard of. Unfamiliar terms and creative code names are all over the place. Making sense of it can be a challenge even for seasoned professionals. And how do you even start exploring the code? Open a random source file?</p> <p>It doesn’t mean that you should not try to understand good open source projects. You should, and the experience can be very rewarding. Yet, it’s not an easy way, and the learning curve will be steep.</p> <p>Code Catalog strives to make learning from other people’s code easier by finding good examples, explaining, annotating and categorizing them.</p> <p>If you find an example interesting, we encourage you to study the original codebase in more detail. You should have a better understanding of how it works by then.</p> <h3 id="who-is-this-project-for"> <a href="#who-is-this-project-for" class="anchor-heading" aria-labelledby="who-is-this-project-for"><svg viewBox="0 0 16 16" aria-hidden="true"><use xlink:href="#svg-link"></use></svg></a> Who is this project for? </h3> <p>Software practitioners and intermediate+ learners of programming.</p> <p>The reader is expected to be able to read non-trivial code in the language used by the example. Most examples expect little domain knowledge and provide context necessary to understand them.</p> <h3 id="what-can-i-learn-from-this"> <a href="#what-can-i-learn-from-this" class="anchor-heading" aria-labelledby="what-can-i-learn-from-this"><svg viewBox="0 0 16 16" aria-hidden="true"><use xlink:href="#svg-link"></use></svg></a> What can I learn from this? </h3> <ul> <li>Style and best practices from prominent open-source projects.</li> <li>Applications of common patterns.</li> <li>Battle-tested solutions to typical problems.</li> </ul> <h3 id="why-so-few-examples"> <a href="#why-so-few-examples" class="anchor-heading" aria-labelledby="why-so-few-examples"><svg viewBox="0 0 16 16" aria-hidden="true"><use xlink:href="#svg-link"></use></svg></a> Why so few examples? </h3> <p>You’d be surprised how much time and effort it takes to find examples meeting our criteria, understand and then explain them. If it was easy, this project would add no value.</p> <p>We are working on adding more. See <a href="https://github.com/ainzzorl/goodcode/issues?q=is%3Aopen+is%3Aissue+label%3A%22new+example%22">proposed examples</a>.</p> <h3 id="who-writes-these-articles"> <a href="#who-writes-these-articles" class="anchor-heading" aria-labelledby="who-writes-these-articles"><svg viewBox="0 0 16 16" aria-hidden="true"><use xlink:href="#svg-link"></use></svg></a> Who writes these articles? </h3> <p>The founder of the project, <a href="https://github.com/ainzzorl">Anton Emelyanov</a>. Future content is expected to be crowd-sourced. Read more about <a href="#contributing">contributing to the project</a>.</p> <p>We believe that the code speaks for itself, though. Our job is to find interesting examples, provide necessary context and point out certain details.</p> <p>Please note that the articles are not written by the authors of the code they are describing. The article author’s understanding of the code can be incomplete or even completely wrong. Your discretion is advised.</p> <h3 id="why-did-you-choose-example-x-from-project-y-and-not-something-else-its-not-nearly-the-most-interesting-thing-in-project-y"> <a href="#why-did-you-choose-example-x-from-project-y-and-not-something-else-its-not-nearly-the-most-interesting-thing-in-project-y" class="anchor-heading" aria-labelledby="why-did-you-choose-example-x-from-project-y-and-not-something-else-its-not-nearly-the-most-interesting-thing-in-project-y"><svg viewBox="0 0 16 16" aria-hidden="true"><use xlink:href="#svg-link"></use></svg></a> Why did you choose <em>Example X</em> from <em>Project Y</em> and not something else? It’s not nearly the most interesting thing in <em>Project Y</em>. </h3> <p>Because we find that <em>Example X</em> satisfies the criteria above. But there’s no reason why there can only be one example from <em>Project Y</em>.
